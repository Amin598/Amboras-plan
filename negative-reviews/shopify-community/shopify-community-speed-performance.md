## Key Patterns

### Checkout speed is a conversion killer
Multiple merchants report 5-20 second checkout loads costing them real sales. Merchants on non-Plus plans cannot even touch the checkout code -- they are completely locked out from fixing this.

### Apps are the #1 speed destroyer
Installing apps degrades performance, and even uninstalling them leaves orphaned code behind in theme files that merchants don't know how to find or remove. One merchant removed all but 5-6 apps and still couldn't get above a 49 score.

### Mobile performance is consistently terrible
Scores in the 20-35 range are described as "normal" for Shopify stores. Multiple merchants report mobile PageSpeed scores under 30, with some as low as 15 or 21.

### The admin panel itself is slow
Not just the storefront -- merchants report 5-10 minute save times in the theme editor and 10-second input lag when typing. Hardware is not the issue; the same machines ran Shopify fine previously.

### Shopify's own infrastructure contributes to the problem
Shopify's internal tracking service (monorail-edge.shopifysvc.com) was identified as a bottleneck that merchants cannot fix, disable, or work around.

### Standard optimizations don't work
Multiple merchants report that the usual advice -- remove apps, compress images, switch themes -- still doesn't fix their speed problems. The underlying platform architecture is the constraint.

### Google PageSpeed scores punish Shopify stores
Even stores with fast actual load times (1.5s) get penalized with scores in the 20s-30s, creating an SEO disadvantage merchants feel helpless about. The bloated JavaScript Shopify injects is the root cause.

### Payment methods slow down checkout
Each additional payment option (Amazon Pay, Google Wallet, etc.) measurably increases checkout load time. Merchants face a tradeoff between offering payment flexibility and having a fast checkout.

## Amboras Implications

Every pattern above maps to a core Amboras advantage:

- **Built-in features instead of apps** = no orphaned code, no third-party script bloat
- **AI-native architecture from day one** = no legacy JavaScript baggage
- **Merchant-controlled checkout** = no locked-down checkout that only Plus plans can touch
- **No internal tracking bloat** = platform analytics that don't slow down the storefront
- **Mobile-first rendering** = sub-second mobile loads as a baseline, not an aspiration
- **Fast admin** = editing your store should feel instant, not like working through mud
