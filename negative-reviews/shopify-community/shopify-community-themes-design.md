## Key Patterns

### Pattern 1: The editor itself is broken
Complaints #2, #3, #4, #5, #8, #9, #10, #11, #12, #13, #21, #22

The theme editor crashes, lags, takes minutes to save, silently loses work, and has gotten worse over time rather than better. The new VS Code-based code editor is slower and more confusing for non-developers. These are not edge cases -- they span 2020-2025 across multiple browsers, devices, and accounts.

**Amboras angle:** A fast, reliable, AI-native editor that never loses work and auto-saves continuously. No distinction between "code editor" and "visual editor" -- one unified experience that works for everyone.

### Pattern 2: Non-coders are locked out of design
Complaints #1, #6, #7, #15, #18

Even basic design tasks (add an image, resize a logo, use a custom font, add an icon) require code or are simply impossible within theme settings. Merchants coming from Squarespace, Wix, or WordPress are shocked by how rigid Shopify themes are. The platform is great for selling but terrible for building.

**Amboras angle:** AI handles the design intent. "Make my logo bigger" or "add our brand font" should be a conversation, not a code commit. True visual design freedom without code.

### Pattern 3: No safe update path
Complaint #17

Theme updates wipe all customizations. No child-theme system, no version control, no merge capability. Merchants with heavily customized themes are trapped -- they can never update without manually re-applying every change.

**Amboras angle:** Built-in version control with non-destructive updates. Customizations live in a separate layer that survives platform and theme updates.

### Pattern 4: Mobile is unreliable
Complaints #19, #20

Theme preview and actual mobile rendering diverge. Themes ship with broken responsive behavior. Apps pile on JavaScript that tanks mobile performance (6-13 second load times, 80% bounce rates).

**Amboras angle:** Mobile-first rendering that is tested against real device behavior, not just a resized browser window. Performance budgets enforced by the platform so apps and themes can't silently kill mobile speed.

### Pattern 5: Hard platform limits with no workarounds
Complaint #16

25 sections per template, 50 blocks per section, 16 blocks per section, 300px logo caps, 100kb liquid code limits. These are arbitrary ceilings that force merchants to find workarounds, hire developers, or accept limitations.

**Amboras angle:** No arbitrary limits on creative expression. If a merchant wants 50 sections, the platform should handle it gracefully rather than saying "not possible."

### Pattern 6: Paid themes still break things
Complaint #14

Even paid themes from the Shopify Theme Store can fail Core Web Vitals, break SEO, and block existing apps. Theme developers may be unable or unwilling to fix fundamental performance issues. Merchants who paid for themes, migration, and customization end up with worse stores than they started with.

**Amboras angle:** Platform-level performance guarantees. Themes that pass the store don't get to break Core Web Vitals. AI-driven performance monitoring that catches regressions before they hit production.
